# NetSuite Automator — Setup Guide

## Prerequisites

- **Node.js** (v18 or later recommended)
- **npm** (comes with Node.js)
- A **NetSuite** account with time-tracking permissions

## Installation

1. Clone or download this repository.

2. Install dependencies:

   ```bash
   npm install
   ```

3. Install Playwright browsers (first time only):

   ```bash
   npx playwright install chromium
   ```

## Configuration

### Option A — Via the Web UI (recommended)

1. Start the app:

   ```bash
   node 3-fill-timesheet.js
   ```

2. The browser opens automatically at `http://localhost:3000`.

3. Click the **⚙ Settings** button in the top-right corner.

4. In the **Credentials** tab, fill in:
   - **Email** — your NetSuite login email
   - **Password** — your NetSuite password
   - **Security Answer** — your NetSuite security-question answer (optional but recommended)

5. Click **Save**. This creates `credentials.json`.

6. In the **Options** tab, add your **Projects** and **Service Items**. You can also use the ✓ button to validate each option against NetSuite.

### Option B — Via the CLI helper

Run the authentication setup script:

```bash
node 1-setup-auth.js
```

This opens a browser window to NetSuite. Log in manually (or automatically if `credentials.json` exists). Once logged in, the session is saved to `auth-state.json` for future use.

### Option C — Manual `credentials.json`

Create a `credentials.json` file in the project root:

```json
{
  "email": "your@email.com",
  "password": "your-password",
  "securityAnswer": "your-security-answer",
  "headless": true
}
```

## Running the App

```bash
node 3-fill-timesheet.js
```

The web UI opens at **http://localhost:3000**. From there you can:

- Add timesheet entries
- Sync entries to NetSuite
- Fetch existing entries from NetSuite
- Edit entries directly in NetSuite
- Manage settings (credentials, options, preferences)

## Standalone Scripts

| Script                    | Purpose                                                 |
| ------------------------- | ------------------------------------------------------- |
| `node 1-setup-auth.js`    | Authenticate and save session cookies                   |
| `node 2-log-time.js`      | Batch-submit entries from `timesheet-data.json` (no UI) |
| `node 4-open-netsuite.js` | Open NetSuite in a browser with saved session           |

## Files Created at Runtime

These files are generated by the app and excluded from version control (`.gitignore`):

| File                    | Contents                             |
| ----------------------- | ------------------------------------ |
| `credentials.json`      | Login credentials and preferences    |
| `auth-state.json`       | Saved browser session (cookies)      |
| `netsuite-entries.json` | Cached entries fetched from NetSuite |

## Troubleshooting

- **"No auth-state.json found"** — Run `node 1-setup-auth.js` or start the web UI and fill in your credentials via Settings.
- **Security question blocks automation** — Add your security answer in Settings → Credentials → Security Answer.
- **Browser not showing during sync/fetch** — Disable "Run browser in background" in Settings → Preferences.
- **Options dropdowns are empty** — Click "Fetch from NetSuite" to auto-populate projects and service items, or add them manually in Settings → Options.
